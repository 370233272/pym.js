Pym.js

Embed iframes responsively

View on GitHub

Released for OpenNews Code Convening, April 2014

About Pym.js

Using iframes in a responsive page can be tricky. It’s easy enough to make an iframe’s width span 100% of its container, but sizing its height is tricky — especially if the content of the iframe changes height depending on page width (for example, because of text wrapping or media queries) or events within the iframe.

Pym.js embeds and resizes an iframe responsively (width and height) within its parent container. It also bypasses the usual cross-domain issues.

Use case: The NPR Visuals team uses Pym.js to embed small custom bits of code (charts, maps, etc.) inside our CMS without CSS or JavaScript conflicts. See an example of this in action.

Requirements/Assumptions

  • The responsive iframe library does not require jQuery or any other libraries. (However, some of the examples use jQuery for unrelated features.)
  • If you’re pasting the iframe embed code into a CMS or blogging software, make sure that it allows you to embed HTML and JavaScript. Some CMSes (such as WordPress) may strip it out.
  • The parent and child pages do not necessarily need to be on the same domain. (If desired, you can restrict this.)

Browsers

This has been tested in:

  • Internet Explorer 9 and 10 (Windows 7)
  • Chrome 32 (Mac 10.9)
  • Firefox 26 (Mac 10.9)
  • Safari 7 (Mac 10.9)
  • iOS 7 Safari
  • Android 4.4 Chrome

Internet Explorer versions earlier than 9 are not supported.

Usage

Resize your browser window to see the responsiveness in action. In all code examples, replace PATH-TO-THIS-FOLDER with the actual published path to your files

On the Parent Page (Where You’re Putting the iFrame)

  • Name your containing div with a unique ID name, and include a data-iframe-target attribute with the path to your child.html file.
  • Include pym.js. Only once per page.
  • Call window.pym.parent(); to set up all responsive iframes on the page. Only once per page.
  • Optional: To filter incoming message domains, you can pass in a regex: window.pym.parent('example', { xdomain: '*\.npr\.org' });
<script type="text/javascript" src="PATH-TO-THIS-FOLDER/pym.js"></script>
<script>window.pym.parent('example', {});</script>

Multiple Embeds on the Same Parent Page

You don’t have to do anything special to the child pages, but you do need to keep a few things in mind for the parent page:

  • Each child embed needs a unique div ID (<div id="example">, <div id="bar-graph">, etc.).
  • Include pym.js only once on the page.
Code Needed on the Parent Page
<div id="example-1" data-iframe-target="PATH-TO-THIS-FOLDER/child-1.html"></div>
<div id="example-2" data-iframe-target="PATH-TO-THIS-FOLDER/child-2.html"></div>
<script type="text/javascript" src="PATH-TO-THIS-FOLDER/pym.js"></script>
<script>
    window.pym.parent('example-1', {});
    window.pym.parent('example-2', {});
</script>

On the Child Page (The Content You Want to Embed)

  • Include pym.js.
  • Invoke window.pym.child();.
  • Optional: If the contents of your iframe are dynamic you will want to pass in a rendering function, like this: window.pym.child({ renderCallback: myFunc }); This function will be called once when the page loads and again any time the window is resized.
  • Optional: You can pass in a number of milliseconds to enable automaticaly updating the height at that rate (in addition to onload and onresize events). For example: window.pym.child({ polling: 500 });.
  • Optional: If you need finer control over resize events, you can invoke window.pym.child.sendHeightToParent() at any time to force the iframe to update its size.

Example: Basic Embed

In our simplest example, we have an HTML table that changes format (via CSS media queries). The height of the iframe adjusts to the height of the content onload and onresize.

Code Needed on the Child Page
<script src="PATH-TO-THIS-FOLDER/pym.js" type="text/javascript"></script>
<script>window.responsiveChild();</script>

Example: Call a Function When the Page Resizes

This might be useful in cases where sections of your child page need to be redrawn based on the new width. (For example, a graphic generated by D3, which would not stretch or reflow on its own.)

Code Needed on the Child Page
<script src="PATH-TO-THIS-FOLDER/pym.js" type="text/javascript"></script>
<script>
function drawGraphic(width) {
...
}
window.responsiveChild({renderCallback: drawGraphic});
</script>

Example: Manual resize events

If you have dynamic content and need finer control over resize events, you can invoke window.responsiveChild.sendHeightToParent() in the child window at any time to force the iframe to update its size. For example, say you have a quiz, and the content of the page changes when someone selects an answer, affecting the page’s height:

Code Needed on the Child Page
<script src="PATH-TO-THIS-FOLDER/pym.js" type="text/javascript"></script>
<script>
function check_answer(e) {
    // highlight the correct answer
    window.responsiveChild.sendHeightToParent();
}
$('.question').find('li').on('click', check_answer);
window.responsiveChild();
</script>

How It Works

The Pym.js library and a small bit of javascript are injected onto the parent page. This code writes an iframe to the page in a container of your choice. The request for the iframe‘s contents includes querystring parameters for the initialWidth and childId of the child page. The initialWidth allows the child to know its size immediately on load, because in iOS, the child frame can not determine its own width accurately. The childId allows multiple children to be embedded on the same page, each with its own communication to the parent.

The child page also includes Pym.js and its own javascript. It initializes cross-iframe communication to the parent, renders any dynamic content and then sends the computed height of the child to the parent via postMessage. Upon receiving this message the parent resizes the containing iframe, thus ensuring the contents of the child are always visible.

The parent page also registers for resize events. Any time one is received, the parent sends the new container width to each child via postMessage. The child re-renders its content and sends back the new height.
